# Vibe Playlist — 최소 기획서

## 서비스 한 줄 정의
매일 아침 기분과 일정을 메모하면, 그날 바이브에 맞는 플레이리스트를 추천해주고 기록으로 남겨주는 서비스.

---

## 타겟 사용자

**"음악은 매일 듣는데 뭘 들을지 고르기 귀찮은 사람"**

- 20–30대, 하루를 음악과 함께 보내는 직장인 / 학생
- Spotify·Apple Music 같은 스트리밍은 쓰지만, 기분에 딱 맞는 플레이리스트를 찾는 과정이 번거로운 사람
- 나중에 "그날 어떤 음악 들었더라"를 돌아보고 싶은 사람

---

## 핵심 기능 (MVP 3가지)

### 1. 오늘의 바이브 메모
- 아침에 짧은 텍스트 입력 (기분, 오늘 일정, 키워드)
- 예시: "오늘 재택, 집중해서 작업해야 함. 약간 피곤"

### 2. 플레이리스트 추천
- 메모를 바탕으로 3–5개의 플레이리스트 카드 표시
- 각 카드: 플레이리스트 이름 + 분위기 태그 + 링크(Spotify/YouTube)
- 추천은 규칙 기반(키워드 → 태그 매핑)으로 시작, 이후 AI 확장 가능

### 3. 플레이리스트 아카이브
- 그날 들은 플레이리스트를 저장
- 날짜별로 메모 + 플레이리스트가 함께 기록됨
- 과거 기록을 달력 또는 리스트 형태로 조회

---

## 사용 흐름

```
아침 메모 입력
    ↓
오늘의 추천 플레이리스트 3–5개 표시
    ↓
마음에 드는 것 선택 → 외부 링크로 재생
    ↓
하루 끝, 들은 플레이리스트 아카이브에 저장
    ↓
과거 기록 조회 (언제 어떤 바이브였는지)
```

---

## MVP 범위 밖 (나중에)

- 소셜 공유
- AI 기반 자동 추천 고도화
- 음악 재생 내장
- 통계/인사이트 ("이번 달 가장 많이 들은 분위기")

---

## 화면 구성 (최소)

| 화면 | 역할 |
|------|------|
| 홈 | 오늘의 메모 입력 + 추천 플레이리스트 |
| 아카이브 | 날짜별 기록 리스트 |
| 기록 상세 | 특정 날의 메모 + 재생했던 플레이리스트 |

---

## 기술 스택 (현재 환경 기준)

- Frontend: React (Vite)
- 데이터 저장: localStorage (MVP), 이후 Supabase 등 확장
- 추천 로직: 키워드 → 분위기 태그 매핑 JSON
